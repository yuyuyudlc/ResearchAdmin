## 1. 系统概述

### 1.1 系统简介

ResearchAdmin（科研项目文档管理系统）是一款专为科研团队设计的一体化协作平台。您可以在系统中：

- **组织项目**：以"工作区"为单位管理多个科研项目
- **管理文档**：在树状文件夹结构中创建、编辑、归档科研文档
- **实时协同**：与团队成员同时在线编辑同一份文档，实时看到彼此的修改
- **精细授权**：以"工作区成员"身份为基础，配合文档级权限控制，实现"谁能看、谁能改、谁能管"的精细管理
- **文件附件**：上传 Word、Excel、PDF 等科研文件，在线预览或下载编辑

### 1.2 浏览器兼容性

| 浏览器 | 最低版本 |
|--------|----------|
| Google Chrome | 90+ |
| Microsoft Edge | 90+ |
| Mozilla Firefox | 90+ |
| Apple Safari | 15+ |

建议使用最新版本的 Chrome 或 Edge 以获得最佳体验。

### 1.3 术语速查

| 术语 | 说明 |
|------|------|
| **工作区 (Workspace)** | 一个科研项目的协作空间，包含文档树和成员 |
| **文档树** | 工作区内文档的层级组织结构（文件夹/子文档） |
| **Owner** | 拥有者，具有最高管理权限的角色 |
| **Member** | 成员，具有基础读写权限的角色 |
| **ACL** | 访问控制列表，用于给特定用户或所有人设置文档级权限 |
| **富文本** | 支持格式化（加粗、标题、列表等）的文档类型 |
| **归档** | 将文档标记为只读冻结状态，常用于项目结题归档 |

## 2. 入门指南

### 2.1 注册账号

1. 打开系统网址，进入登录页面
2. 点击 **"注册"** 链接切换到注册模式
3. 填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 用户名 | 登录时使用的名称 | ✅ |
| 邮箱 | 用于登录和找回密码 | ✅ |
| 密码 | 不少于 6 位 | ✅ |
| 所属组织 | 您所在的课题组或实验室名称 | 选填 |
| 职称 | professor / lecturer / researcher / engineer 等 | 选填 |
| 导师 | 您的导师姓名（研究生适用） | 选填 |
| 头像链接 | 头像图片的 URL | 选填 |
| 个性签名 | 展示在个人资料中的一句话 | 选填 |

4. 点击 **"注册"** 按钮
5. 注册成功后系统会自动为您创建一个 **"我的私人空间"** 工作区

### 2.2 登录系统

1. 在登录页面输入您的 **邮箱** 和 **密码**
2. 点击 **"登录"** 按钮
3. 验证通过后自动跳转到系统首页

> **提示**：登录有效期为 24 小时，超时后需重新登录。

### 2.3 系统界面概览

```
┌──────────────────────────────────────────────┐
│  左侧导航栏          │   主内容区域           │
│                      │                       │
│  🏠 首页             │   [当前页面内容]       │
│  📁 工作区           │                       │
│  ⚙ 管理后台         │                       │
│  (管理员可见)        │                       │
│                      │                       │
│  ─────────           │                       │
│  用户头像            │                       │
│  退出登录            │                       │
└──────────────────────────────────────────────┘
```

- **左侧导航栏**：切换不同功能页面
- **主内容区域**：展示当前页面的操作界面
- **用户头像区域**：查看个人资料、退出登录

## 3. 个人账户管理

### 3.1 查看与编辑个人资料

1. 点击左侧导航栏底部的用户头像区域
2. 进入个人资料页面
3. 可修改以下信息：
   - 用户名
   - 邮箱
   - 所属组织
   - 职称
   - 导师
   - 头像链接
   - 个性签名
4. 修改完成后点击 **"保存"**

### 3.2 修改密码

1. 在个人资料页面找到"修改密码"入口
2. 输入 **旧密码**
3. 输入 **新密码**（不少于 6 位）
4. 再次确认新密码
5. 点击 **"确认修改"**
6. 修改成功后，下次登录需使用新密码

### 3.3 切换头像与签名

在个人资料编辑页面直接修改：
- **头像**：填入图片 URL 链接
- **签名**：填写一行话（如"专注科研文档协作与知识管理"）

## 4. 工作区管理

### 4.1 什么是工作区

工作区是一个科研项目的协作空间。每个工作区包含：
- **文档树**：该项目下的所有文档和文件夹
- **成员列表**：可以访问该项目的团队成员
- **权限设置**：成员角色（Owner/Member）

> 例如：您可以创建"深度学习科研组"工作区来管理该项目相关的所有文档。

### 4.2 查看我的工作区

1. 点击左侧导航栏 **"工作区"**
2. 系统展示您参与的所有工作区列表
3. 每个工作区卡片显示名称、描述和您的角色

### 4.3 创建新工作区

1. 在工作区列表页面，点击 **"创建工作区"** 按钮
2. 在弹出的对话框中填写：
   - **工作区名称**（必填）：如"论文撰写项目"
   - **描述**（选填）：如"存放论文相关的所有文献与草稿"
3. 点击 **"确定"**
4. 创建成功后，您自动成为该工作区的 **Owner**

### 4.4 编辑工作区信息

1. 在工作区列表中找到目标工作区
2. 点击 **"编辑"** 按钮（仅 Owner 可见）
3. 修改名称或描述
4. 点击 **"保存"**

### 4.5 管理成员

> 仅工作区 **Owner** 可管理成员。

#### 4.5.1 添加成员

1. 进入目标工作区详情页
2. 找到 **"成员管理"** 区域
3. 搜索需要添加的用户（输入用户名或邮箱）
4. 选择目标用户，设置角色：
   - **Owner**：管理员，有完全控制权
   - **Member**：成员，有基础读写权限
5. 点击 **"添加"**

#### 4.5.2 修改成员角色

1. 在成员列表中找到目标成员
2. 点击 **"修改角色"**
3. 在 Owner 和 Member 之间切换
4. 点击 **"确认"**

> **注意**：工作区必须至少保留一名 Owner，不能将所有 Owner 降级或移除。

#### 4.5.3 移除成员

1. 在成员列表中找到目标成员
2. 点击 **"移除"**
3. 在弹出的确认框中点击 **"确认移除"**

> **注意**：如果该成员是最后一名 Owner，系统将拒绝移除。

### 4.6 删除工作区

1. 在工作区列表中找到目标工作区
2. 点击 **"删除"** 按钮（仅 Owner 可见）
3. 在确认框中点击 **"确认删除"**
4. 工作区状态变为"已删除"，不再显示在列表中

> ⚠️ **警告**：删除工作区将导致该工作区内所有文档不再可访问。请谨慎操作。

## 5. 文档管理

### 5.1 进入工作区文档树

1. 在工作区列表中点击目标工作区
2. 进入工作区详情页，左侧为文档树，右侧为文档详情区
3. 文档树以层级结构展示：
   - 📁 文件夹（有子文档的父文档）
   - 📄 文档（叶子节点）
4. 点击文件夹左侧箭头展开/折叠子文档

### 5.2 创建新文档

1. 在工作区详情页，点击 **"新建文档"** 按钮
2. 选择文档类型：
   - **富文本文档**：可在线编辑的格式化文档
   - **文件附件**：上传 Word/Excel/PDF 等文件
3. 填写文档标题
4. （可选）填写文档摘要
5. （可选）选择父文件夹（不选则创建在根目录）
6. 点击 **"创建"**

### 5.3 创建文件夹（父文档）

与创建普通文档相同，系统会根据 `parentId` 关系自动形成树结构：
1. 先创建根目录下的文档作为"文件夹"
2. 再在创建新文档时选择该文档作为父节点

### 5.4 编辑文档标题与摘要

1. 点击打开目标文档
2. 在编辑器顶部找到标题栏
3. 修改标题或摘要
4. 点击 **"保存"** 更新元数据

### 5.5 移动文档

1. 打开目标文档
2. 点击 **"移动"** 按钮（需 Manage 权限）
3. 在弹出的对话框中：
   - 选择目标父文件夹
   - 设置排序顺序（可选）
4. 点击 **"确认移动"**

> ⚠️ **限制**：
> - 不能将文档移动到自己的子文档下（防止循环引用）
> - 不能跨工作区移动文档

### 5.6 归档与恢复文档

#### 归档

当文档不再活跃使用时（如项目结题），可将其归档：
1. 打开目标文档
2. 点击 **"归档"** 按钮（需 Manage 权限）
3. 确认操作
4. 文档状态变为"已归档"，此时文档为只读，不可编辑

#### 恢复

如需重新使用已归档的文档：
1. 找到已归档的文档
2. 点击 **"恢复"** 按钮
3. 文档状态恢复为"活跃"，可正常编辑

### 5.7 删除文档

1. 打开目标文档
2. 点击 **"删除"** 按钮（需 Manage 权限）
3. 在弹出的确认框中点击 **"确认删除"**
4. 文档变为"已删除"状态，不再出现在文档树中

> ⚠️ **注意**：删除操作会同时清理关联的 ACL 规则和正文数据。

1. **实时内容同步**：他人的编辑内容会实时出现在您的屏幕上
2. **光标显示**：您可以看到其他协作者的光标位置，标注了对方的名字
3. **在线状态**：编辑器顶部显示当前在线协作者头像列表
4. **无需手动刷新**：所有更改自动同步

> **注意**：协同编辑功能需要 WebSocket 连接正常工作。如遇到连接问题请刷新页面。

### 6.4 保存与自动保存

- 系统在您编辑时自动保存内容，无需手动点击保存按钮
- 编辑器底部状态栏显示保存状态：
  - "已保存"：内容已持久化到服务器
  - "保存中..."：内容正在传输
- 关闭页面前请确认状态为"已保存"

## 7. 文档共享与权限管理

### 7.1 权限模型简介

系统的权限体系分为两层：

| 层级 | 说明 |
|------|------|
| **工作区层面** | 工作区 Owner 和 Member 具有基础权限 |
| **文档层面 (ACL)** | 可为特定用户或所有人设置独立权限 |

最终权限取两者的最高优先级组合（Owner > 显式 ACL > Public > 工作区默认）。

**权限位说明**：

| 权限 | 含义 |
|------|------|
| **只读 (Read)** | 可以查看文档内容和评论 |
| **编辑 (Edit)** | 可以编辑内容、发表评论 |
| **管理 (Manage)** | 可以修改权限、归档、删除文档 |
| **拒绝 (Deny)** | 完全禁止访问（最高优先级） |

### 7.2 打开权限分享弹窗

1. 打开目标文档
2. 点击 **"权限设置"** 或 **"分享"** 按钮
3. 弹出 ACL（访问控制列表）管理弹窗

弹窗中展示当前文档的所有显式权限规则。

### 7.3 添加用户授权

1. 在 ACL 弹窗中点击 **"添加权限"**
2. 在搜索框中输入对方的**用户名**、**邮箱**或**显示名称**
3. 从下拉列表中选中目标用户
4. 设置权限等级：
   - 只读 (1)：对方只能查看
   - 编辑 (2)：对方可编辑内容
   - 编辑+管理 (6)：对方可编辑和管理权限
   - 完全控制 (7)：对方拥有所有权限
   - 拒绝 (8)：禁止对方访问
5. 可选：勾选 **"允许继承"**，使子文档自动继承此权限规则
6. 点击 **"确定"** 添加

### 7.4 设置公开访问

如需让所有用户都能访问该文档：
1. 在 ACL 弹窗中点击 **"添加权限"**
2. 选择 **"所有人 (Public)"**
3. 设置权限（通常设为"只读"）
4. 点击 **"确定"**

> ⚠️ **注意**：公开访问意味着系统中所有注册用户都可以按给定权限访问此文档。

### 7.5 修改或撤销授权

1. 在 ACL 弹窗中找到要修改的规则
2. 点击对应规则旁边的 **"编辑"** 或 **"删除"** 按钮
3. 编辑模式：修改权限等级后保存
4. 删除模式：确认后规则被移除，对应授权即刻失效

### 7.6 权限继承说明

- 当文档设置了 **"允许继承"**（inherit=true）时，其子文档会自动继承该权限规则
- 如果子文档同时有自己的显式 ACL 规则，则以子文档自身规则为准
- 如果子文档清除了继承标记，则不会继续向更深层传递

> **示例**：在"文献综述"文件夹设置了"张三 只读 + 允许继承"，那么"文献综述"下的所有文档都默认让张三可以查看。

## 12. 常见问题 FAQ

### Q1: 忘记密码怎么办？
A: 请联系系统管理员为您重置密码。重置后使用新密码登录，并建议立即修改为个人密码。

### Q2: 为什么我打开的文档无法编辑？
A: 可能原因：
- 您对该文档仅有**只读**权限，联系文档 Owner 申请编辑权限
- 文档已被**归档**，处于只读冻结状态。如需编辑请先"恢复"文档

### Q3: 多人同时编辑会不会互相覆盖？
A: 不会。系统采用 CRDT 实时协同技术，多人编辑自动合并，不会丢失任何人的内容。

### Q4: 如何邀请外部合作者加入工作区？
A: 使用工作区 Owner 身份，在成员管理页面搜索并添加用户。被邀请者需先在系统中注册。

### Q5: 如何让非工作区成员也能查看某个文档？
A: 打开文档的 ACL 权限设置，通过"添加权限"为特定用户设置只读权限，或设置为"公开只读"。

### Q6: 文档删除了能找回吗？
A: 系统使用软删除机制，删除的文档标记为"deleted"状态。如需恢复，请联系管理员进行数据恢复。

### Q7: 支持哪些文件格式的预览？
A: 当前支持：Word (.docx)、Excel (.xlsx)、PowerPoint (.pptx)、PDF (.pdf)。其他格式需要下载后本地查看。

### Q8: 协同编辑时我的光标别人能看到吗？
A: 可以。协同编辑功能打开后，协作者之间可以看到彼此的光标位置和用户名。

### Q9: 工作区的 Owner 可以有几个？
A: 可以有多个 Owner，但至少保留一个。系统会阻止移除或降级最后一名 Owner。

### Q10: 系统对文件大小有限制吗？
A: 建议单个文件不超过 50MB。超大文件请使用独立存储方案。
